# Versionado y Canales

## Estrategia de versionado

Formato: `x.y.z`

- `x`: cambios grandes / arquitectura / UX importante.
- `y`: subida de version de Tailscale u otro salto funcional relevante.
- `z`: correcciones y mejoras incrementales.

## Canales

- **Estable**: versiones validadas para la mayoria de usuarios.
- **Beta**: versiones preliminares para pruebas y feedback rapido.

Flujo recomendado:

1. Implementar en beta.
2. Validar con logs y pruebas reales.
3. Promocionar a estable cuando quede pulido.
